2 Session Lifecycle

Two paths: ad-hoc (CC hooks) and managed (TUI-initiated). Both converge at the registry. 

CC starts  ──▶  SessionStart hook  ──▶  nexus-register start
                                           │
                                           │ gRPC RegisterSession
                                           ▼
                                     nexus-agent
                                           │
                                           │ registry.register_adhoc()
                                           ▼
                                     SessionRegistry  ──▶  SessionStarted event
                                           │
CC tool use  ─▶  PostToolUse hook  ──▶  nexus-register heartbeat
                                           │
                                           │ gRPC Heartbeat
                                           ▼
                                     registry.heartbeat()  ──▶  HeartbeatReceived event
                                                                 (+ StatusChanged if Stale→Active)
                                           │
CC stops  ───▶  Stop hook  ──────────▶  nexus-register stop
                                           │
                                           │ gRPC UnregisterSession
                                           ▼
                                     registry.unregister()  ──▶  SessionStopped event

TUI press n  ──▶  StartSession RPC  { project, cwd }
                         │
                         ▼
                   nexus-agent
                         │
                   1. Generate UUID
                   2. Session { pid: 0, cc_session_id: uuid }
                   3. registry.register_managed()  ──▶  SessionStarted
                   4. Bootstrap:
                         │
                         │  claude -p --session-id <uuid>
                         │    --output-format stream-json --verbose
                         │    --dangerously-skip-permissions
                         │    "Acknowledge with: Ready."
                         │
                         ▼
                   Session resumable via --resume <uuid>

every 30s:

  for each session where tmux_session.is_none():

    idle_seconds() > 900s (15 min)  ──▶  REMOVE  + emit SessionStopped
                                             "stale session removed"

    idle_seconds() > 300s (5 min)   ──▶  STALE   + emit StatusChanged
                                             Active → Stale

    heartbeat arrives on Stale       ──▶  REVIVE  + emit StatusChanged
                                             Stale → Active

3 Command Broker Flow

User types a prompt in the TUI → agent spawns CC subprocess → output streams back. 

TUI                          Agent                         CC subprocess
 │                              │                               │
 │ SendCommand                 │                               │
 │  { session_id, prompt }     │                               │
 ├──────────────────────────▶  │                               │
 │                              │ 1. heartbeat(session_id)     │
 │                              │ 2. lookup session in registry│
 │                              │ 3. resolve resume_id / cwd  │
 │                              │                               │
 │                              │ spawn process:                │
 │                              │  NEXUS_SUBPROCESS=1          │
 │                              │  claude -p                   │
 │                              │   --resume <id> (managed)    │
 │                              │   --session-id <new> (adhoc) │

 │◀─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │◀─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─│
 │                              │                               │
 │ input bar re-enabled        │                               │
 │                              │                               │
NEXUS_SUBPROCESS=1 — prevents the CC subprocess from firing nexus hooks (which would register/unregister itself and corrupt the parent session). All three hooks (start, heartbeat, stop) check this var and exit immediately.

Session {
  id              : String        // UUID, primary key
  pid             : u32           // OS PID; 0 = managed, >0 = ad-hoc
  project         : Option        // "oo", "nx", etc.
  cwd             : String        // working directory
  branch          : Option        // git branch (not populated yet)
  started_at      : Timestamp     // creation time
  last_heartbeat  : Timestamp     // updated by Heartbeat RPC
  status          : enum          // Active | Idle | Stale | Errored
  cc_session_id   : Option        // CC UUID for --resume
  tmux_session    : Option        // legacy, always None now
  spec            : Option        // openspec name (not populated)
  command         : Option        // initial command (not populated)
  agent           : Option        // agent name (not populated)
}

CommandOutput .content oneof {

  TextChunk    { text, partial }       // assistant response text
                                          // partial=true for streaming tokens

  ToolUseInfo  { tool_name,             // "Bash", "Read", "Edit" etc.
                 input_preview }        // first 200 chars of tool input

  ToolResult   { tool_name,             // tool that ran
                 output_preview,        // first 500 chars of output
                 success }              // true/false

  CommandError { message,               // error description
                 exit_code }            // CC process exit code

  CommandDone  { duration_ms,           // total command time
                 tool_calls }           // num_turns from CC
}

SessionEvent .payload oneof {

  SessionStarted     { session: Session }
  HeartbeatReceived  { last_heartbeat: Timestamp }
  StatusChanged      { old_status, new_status }
  SessionStopped     { reason: String }
}
No persistence — the registry is an in-memory RwLock<HashMap>. Sessions are lost on agent restart. Ad-hoc sessions self-heal via the next CC heartbeat hook. Managed sessions require a new StartSession from the TUI.
